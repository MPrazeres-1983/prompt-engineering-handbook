# Contributing

Contributions are welcome, especially:
- new templates with clear constraints and intended usage
- examples with inputs/outputs + evaluation notes
- improvements to methodology and rubric

## Contribution rules
- Keep content practical and reproducible
- Avoid marketing language and unverifiable claims
- Include at least one example and one evaluation note for new templates

## How to propose changes
1. Fork the repo
2. Create a branch: `feature/<short-name>`
3. Open a PR explaining:
   - what you changed
   - why it improves repeatability or safety
   - how to validate it
